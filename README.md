# convex-starter

A house scaffold: **Vite + React 19 (React Compiler) + Convex + Convex Auth
(Google + optional GitHub) + TanStack Router (file-based) + Tailwind v4**. No
product logic — just the conventions, wiring, and tooling shared across
projects.

> This repo also hosts the **`convex-starter` skill**
> (`.claude/skills/convex-starter/`). The repo itself is the template the skill
> copies to bootstrap new projects — no duplicated snapshot.

## Setup

1. `pnpm install`
2. `pnpm exec convex dev` — creates a Convex project and generates
   `convex/_generated/`. (Or just run `pnpm dev`, which runs Convex + Vite
   together.)
3. **Convex Auth:**
   - Run `pnpm exec @convex-dev/auth` to generate the JWT keys, or follow
     <https://labs.convex.dev/auth>.
   - Set `AUTH_GOOGLE_ID` + `AUTH_GOOGLE_SECRET` in the Convex dashboard.
   - GitHub is **optional** — set `AUTH_GITHUB_ID` + `AUTH_GITHUB_SECRET` to
     enable the GitHub button.
4. Copy `.env.local.example` → `.env.local` and set `VITE_CONVEX_URL`.
5. `pnpm dev`

> First run generates `src/routeTree.gen.ts` (TanStack Router — gitignored) and
> `convex/_generated/` (Convex — committed, regenerated by `convex dev`).

## Scripts

| Script                        | What                           |
| ----------------------------- | ------------------------------ |
| `pnpm dev`                    | Convex + Vite concurrently     |
| `pnpm build`                  | `tsc -b && vite build`         |
| `pnpm lint` / `pnpm lint:fix` | ESLint (+ Convex plugin)       |
| `pnpm fix`                    | Prettier write + ESLint fix    |
| `pnpm knip`                   | Unused files / deps / exports  |
| `pnpm clone`                  | Copy prod Convex data into dev |

## Make it yours (delete the example)

- Remove the `notes` table (`convex/schema.ts`), `convex/notes.ts`, and the
  notes UI in `src/routes/index.tsx`.
- Rename the app in `package.json` and `index.html` (title / OG / theme-color),
  and drop real favicons into `public/`.
- Adjust the dev port in `vite.config.ts` and the Plausible domain in
  `index.html`.
